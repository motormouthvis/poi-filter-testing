Dev brief: hospitals POI — dedupe, under-tags, new fields, evaluation
1) Goals
* Dedupe: one user-facing pin per same physical site (not “same health system”).
* Under-tags: include real acute hospitals when Overture has category_alternate empty but the site is still a hospital (e.g. Martin South).
* Nationwide: avoid city/name whitelists; prefer DB-owned fields + optional external verification.
________________


2) Dedupe rules (implement + test)
* Merge candidates only if normalized street address matches (number + street + unit rules + city + state; normalize St/Street, etc.) OR haversine distance ≤ 50–100 m and same operator/brand if available.
* Do not merge different addresses (e.g. main hospital vs freestanding ER miles away).
* Do merge obvious duplicates: same address/coords “Hospital” + “Emergency Department” rows.
* Output: canonical_poi_id (or keep one row, mark others duplicate_of_id).
________________

4) Population pipeline (cron/batch)
Step A — classify overture_tagged (cheap)
If existing fields already satisfy your strict rule (e.g. alternate contains emergency_room / medical_center without denylist hits), set:
* acute_hospital_er_capable = true
* acute_hospital_source = overture_tagged
Step B — verified_external (preferred for fixes like Martin South)
Match POI → external record(s) with documented match keys (name + address + distance threshold + optional brand). On confident match that indicates ED/acute hospital, set:
* acute_hospital_er_capable = true
* acute_hospital_source = verified_external
* store evidence ids in acute_hospital_evidence
Step C — inferred_internal (fallback, conservative)
Only when not overture_tagged and not verified_external, consider rules such as:
* basic_category/category_primary indicates hospital AND
* category_alternate is null/empty AND
* confidence ≥ tuned threshold AND
* business name not hit short denylist AND
* if alternates exist, must not hit alternate denylist
Set inferred_internal + boolean; log all true positives/false positives for review.
Step D — manual overrides
Rare corrections via admin only; manual_override wins in merge order.
Processing order: manual_override > verified_external > overture_tagged > inferred_internal > else false/unknown.
________________


5) Evaluation (must ship with metrics)
Build a labeled set (start with 200–500 pins across ≥5 metros): each nearby hospital POI = keep/drop for “ER-capable acute hospital layer”.
Track per run:
* Precision@N / FP rate in top results
* Recall vs labeled “must show” list (include Martin South–type cases)
* Counts by acute_hospital_source
* Table QA: top FPs by category_alternate, taxonomy, confidence, name token hits
Release gate: FP regression budget (e.g. “no more than X FP per 100 pins”) + no loss of labeled must-shows.
